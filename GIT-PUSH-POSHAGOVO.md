# Пошагово: подключить папку к GitHub и включить автодеплой на REG.RU

После этих шагов при каждом **push** в GitHub файлы будут автоматически заливаться на optikadobryhcen.ru.

---

## Шаг 1. Узнать адрес репозитория на GitHub

1. Откройте в браузере **github.com** и войдите в аккаунт.
2. Откройте **репозиторий** с сайтом «Оптика добрых цен» (тот, откуда идёт деплой на REG.RU).
3. Нажмите зелёную кнопку **«Code»**.
4. Скопируйте адрес. Он будет таким:
   - **HTTPS:** `https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git`
   - или **SSH:** `git@github.com:ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git`

Запомните или вставьте в блокнот — понадобится в шаге 3.

---

## Шаг 2. Открыть «Терминал» на Mac

1. Нажмите **Cmd + Пробел**, введите **Терминал** (или **Terminal**).
2. Нажмите **Enter** — откроется окно с чёрным (или белым) фоном и строкой ввода.

---

## Шаг 3. Перейти в папку проекта

В Терминале введите (можно вставить целиком) и нажмите **Enter**:

```bash
cd /Users/ekaterinamacaberidze/Documents/optika-site-static
```

Должна появиться строка без ошибки. Теперь все команды выполняются из этой папки.

---

## Шаг 4. Проверить, что remote ещё не добавлен

Введите и нажмите **Enter**:

```bash
git remote -v
```

Если вывод **пустой** — remote нет, можно добавлять. Если уже есть строка с `origin` и адресом GitHub — переходите к шагу 6 (просто сделать push).

---

## Шаг 5. Добавить GitHub как remote

Вместо `ВАШ_ЛОГИН` и `ИМЯ_РЕПОЗИТОРИЯ` подставьте свои данные из шага 1.

**Если копировали HTTPS-адрес:**

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
```

**Пример:** если репозиторий `https://github.com/ivanov/optika-site`, то команда:

```bash
git remote add origin https://github.com/ivanov/optika-site.git
```

Нажмите **Enter**. Ошибка «remote origin already exists» значит, что remote уже есть — тогда шаг 5 пропустить и перейти к шагу 6.

---

## Шаг 6. Отправить код на GitHub (push)

Введите и нажмите **Enter**:

```bash
git push -u origin main
```

- Может появиться запрос **логина и пароля**.  
  На GitHub для HTTPS сейчас используется не пароль, а **Personal Access Token (токен)**.  
  Если просит пароль — введите токен (создать: GitHub → Settings → Developer settings → Personal access tokens).
- Если включён **SSH-ключ** и репозиторий добавлен по SSH — запроса может не быть.

После успешного выполнения в конце будет что-то вроде:  
`main -> main` или `Branch 'main' set up to track remote branch 'main'`.

---

## Шаг 7. Проверить автодеплой

1. Откройте на GitHub вкладку **Actions** вашего репозитория.
2. Должен появиться новый запуск workflow **«Deploy to REG.RU»** (после вашего push).
3. Подождите 1–2 минуты, пока статус станет зелёным (галочка).
4. Откройте **https://optikadobryhcen.ru** и сделайте жёсткое обновление (**Cmd + Shift + R**) или откройте в режиме инкогнито.
5. Проверьте: правой кнопкой → «Просмотреть код страницы» → поиск (**Cmd + F**) по **ver:2025-03-12**. Если строка есть — на сайте уже новая версия с GitHub.

---

## Дальше: как обновлять сайт

1. Меняете файлы в папке **optika-site-static** на Mac (в Cursor или любом редакторе).
2. В Терминале из этой папки выполняете:

```bash
cd /Users/ekaterinamacaberidze/Documents/optika-site-static
git add .
git commit -m "Кратко: что изменили"
git push
```

3. Через 1–2 минуты workflow зальёт изменения на REG.RU — сайт обновится автоматически.

---

## Если что-то пошло не так

| Проблема | Что сделать |
|----------|-------------|
| `remote origin already exists` | Remote уже добавлен. Выполните только `git push -u origin main` (шаг 6). |
| `Permission denied` / `Authentication failed` | Использовать токен вместо пароля (GitHub → Settings → Developer settings → Personal access tokens) или настроить SSH-ключ. |
| `failed to push` / `rejected` | На GitHub есть коммиты, которых нет у вас. Выполните: `git pull origin main`, затем снова `git push -u origin main`. |
| Ветка на GitHub называется не `main`, а `master` | В шаге 6 заменить `main` на `master`: `git push -u origin master`. |

Если напишете, на каком шаге остановились и какой текст ошибки видите, можно разобрать точечно.
